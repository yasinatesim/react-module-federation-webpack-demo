# Module Federation Demo — Design Spec

**Date:** 2026-04-19  
**Project:** react-module-federation-demo  
**Status:** Approved

## Overview

A minimal, article-faithful demo of Module Federation 2.0 consisting of two independent React applications: `shell-app` (host) and `products-app` (remote). The demo accompanies a Turkish blog article on Module Federation and must not go beyond what is described in that article.

## Application Details

### products-app (Remote, port 3001)

**webpack.config.js key config:**
- `name: 'products_app'`
- `filename: 'remoteEntry.js'`
- `exposes: { './ProductList': './src/ProductList' }`
- `shared: { react: { singleton: true, requiredVersion: '^18.0.0' }, 'react-dom': { singleton: true, requiredVersion: '^18.0.0' } }`
- `output.publicPath: 'auto'`

**ProductList.jsx:** Static list of two products (Mekanik Klavye 1200₺, Ergonomik Mouse 450₺) — exactly as in the article.

**Async boundary pattern:**
- `src/index.js` → `import('./bootstrap')`
- `src/bootstrap.js` → mounts `<ProductList />` to `#root`

### shell-app (Host, port 3000)

**webpack.config.js key config:**
- `name: 'shell'`
- `remotes: { products_app: 'products_app@http://localhost:3001/remoteEntry.js' }`
- Same `shared` config as products-app
- `output.publicPath: 'auto'`

**App.jsx:** Uses `React.lazy` + `Suspense` + a custom inline `ErrorBoundary` class component to load `products_app/ProductList`. `ErrorBoundary` is defined in `src/ErrorBoundary.jsx`.

**Async boundary pattern:** Same as products-app.

## Constraints

- Demo must stay strictly within the article's scope — no additional remotes, no routing, no state management.
- `publicPath: 'auto'` used instead of hardcoded localhost URLs for safer builds.
- `ErrorBoundary` wraps the remote component in shell-app to prevent full app crashes on remote load failure.
